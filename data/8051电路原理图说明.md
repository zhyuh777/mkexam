# 8051 单片机电路原理图说明

本文档对四张 8051 系列单片机实验/应用电路原理图进行说明，涵盖主要元器件、引脚连接关系及编程要点，便于编写相应控制程序。

---

## 图1：80C51 单片机 LED 闪烁电路

![80C51 LED 闪烁电路](img/image-f00f0932-2ad6-47c0-9fa8-1358803a1df3.png)

### 电路概述

本电路为最基本的 8051 最小系统扩展，在 **80C51（U1）** 上连接一只红色 LED，用于演示 I/O 口输出控制。

### 主要元器件

| 编号 | 型号/类型 | 作用 |
| :--- | :--- | :--- |
| U1 | 80C51 | 8 位单片机，系统核心 |
| RP1 | RESPACK-8 | 8 路排阻，为 P0 口提供上拉 |
| D1 | LED-RED | 红色发光二极管，指示输出 |

### 引脚连接

**Port 0 与上拉电阻（RP1）**

- RP1 第 1 脚接 **VCC**
- RP1 第 2 脚接 **P0.0**（U1 第 39 脚）
- P0 口为开漏输出，需外部上拉电阻才能输出高电平

**LED（D1）**

- 阳极：接 **P0.0** 与 RP1 第 2 脚的公共节点
- 阴极：接 **GND**

**其他引脚**

- P0.1～P0.7：经 RP1 上拉（图中仅 P0.0 外接 LED）
- P1、P2、P3 及 XTAL1/XTAL2、RST、PSEN、ALE、EA 等：按最小系统常规连接，图中未作额外外设扩展

### 工作逻辑

| P0.0 状态 | LED 状态 | 说明 |
| :--- | :--- | :--- |
| 高电平（1） | 亮 | 引脚高阻，RP1 上拉提供电流经 LED 到地 |
| 低电平（0） | 灭 | 引脚拉低，电流经 P0.0 到地，LED 无足够驱动电流 |

### 编程要点

- 控制位：`P0^0` 或 `P0.0`
- 实现 LED 闪烁：在循环中交替置 1/置 0，并插入适当延时
- C51 示例思路：`sbit LED = P0^0;`，循环中 `LED = 1; delay(); LED = 0; delay();`

## 图2：AT89C51 驱动六位七段数码管电路

![六位七段数码管电路](img/image-5a5bf199-4d1e-470f-8c25-8817daf370fa.png)

### 电路概述

本电路使用 **AT89C51（U1）** 配合 **74HC573（U2）** 锁存器，驱动 **6 位共阴/共阳七段数码管**（动态扫描显示）。段码由 P0 口输出，位选由 P2 口经锁存器保持。

### 主要元器件

| 编号 | 型号/类型 | 作用 |
| :--- | :--- | :--- |
| U1 | AT89C51 | 单片机，输出段码与位选数据 |
| U2 | 74HC573 | 八位透明锁存器，锁存位选信号 |
| RP1 | RESPACK-8 | P0 口上拉电阻 |
| — | 6 位七段数码管 | 显示输出 |

### 引脚连接

**Port 0 — 段码数据总线**

| MCU 引脚 | 数码管段 |
| :--- | :--- |
| P0.0（39） | A |
| P0.1（38） | B |
| P0.2（37） | C |
| P0.3（36） | D |
| P0.4（35） | E |
| P0.5（34） | F |
| P0.6（33） | G |
| P0.7（32） | DP（小数点） |

P0.0～P0.7 同时经 RP1 接 VCC 上拉。

**Port 2 — 位选数据输入（至 74HC573）**

| MCU 引脚 | 74HC573 输入 | 锁存器输出 | 数码管位 |
| :--- | :--- | :--- | :--- |
| P2.0（21） | D0（2） | Q0（19） | 第 1 位 |
| P2.1（22） | D1（3） | Q1（18） | 第 2 位 |
| P2.2（23） | D2（4） | Q2（17） | 第 3 位 |
| P2.3（24） | D3（5） | Q3（16） | 第 4 位 |
| P2.4（25） | D4（6） | Q4（15） | 第 5 位 |
| P2.5（26） | D5（7） | Q5（14） | 第 6 位 |

**Port 1 — 锁存控制**

- **P1.0（1）** → 74HC573 **LE**（Latch Enable，11 脚）
  - LE = 1：输出随输入变化（透明）
  - LE 由高变低：锁存当前输入数据

**74HC573 其他控制脚**

- **OE**（1 脚）接 **GND**：输出始终有效

### 动态扫描显示流程

1. 向 **P0** 写入当前位的段码（如显示数字 "0" 的编码）
2. 向 **P2** 写入位选掩码（如 `0x01` 选中第 1 位）
3. **P1.0** 先置高再置低，产生锁存脉冲，将位选数据锁入 74HC573
4. 短暂延时后，关闭当前位、切换下一位，重复上述步骤
5. 6 位循环扫描频率通常 **> 60 Hz**，利用视觉暂留实现无闪烁显示

### 编程要点

- 需准备 0～9（及可能的 A～F）段码表，按共阴/共阳极性取反
- 位选与段码时序配合：先送段码，再锁存位选，避免鬼影
- 锁存操作：`P1.0 = 1; P1.0 = 0;` 或等价脉冲

## 图3：AT89C51 两位数码管与 4×4 矩阵键盘电路

![两位数码管与矩阵键盘电路](img/image-25b37314-a825-472f-b090-7cacbffb1d67.png)

### 电路概述

本电路在 **AT89C51（U1）** 上同时接入 **2 位七段数码管**（输出）和 **4×4 矩阵键盘**（输入）。数码管段码由 P0 驱动，位选经 **74HC04（U2）** 反相器控制；键盘矩阵接 P1 口。

### 主要元器件

| 编号 | 型号/类型 | 作用 |
| :--- | :--- | :--- |
| U1 | AT89C51 | 单片机 |
| U2 | 74HC04 | 六反相器，反相位选信号 |
| RP1 | RESPACK-8 | P0 口上拉电阻 |
| — | 2 位七段数码管 | 显示输出 |
| — | 4×4 矩阵键盘 | 16 键输入 |

### 引脚连接

**Port 0 — 段码（与图2相同）**

| MCU 引脚 | 数码管段 |
| :--- | :--- |
| P0.0～P0.6 | A～G |
| P0.7 | DP |

**Port 2 — 位选（经 74HC04 反相）**

| MCU 引脚 | 74HC04 | 数码管公共端 |
| :--- | :--- | :--- |
| P2.0（21） | U2:A 输入 → 输出 | 第 1 位（'1'） |
| P2.1（22） | U2:B 输入 → 输出 | 第 2 位（'2'） |

**反相逻辑说明**

- P2.x = **高电平** → 经反相器输出 **低电平** → 对应位 **选中**（共阴数码管）
- P2.x = **低电平** → 输出 **高电平** → 对应位 **关闭**

**Port 1 — 4×4 矩阵键盘**

| 引脚 | 键盘连接 | 扫描角色 |
| :--- | :--- | :--- |
| P1.0～P1.3（1～4） | 行线 Row0～Row3 | 通常作输出，逐行拉低 |
| P1.4～P1.7（5～8） | 列线 Col0～Col3 | 通常作输入，读列状态 |

### 键盘扫描原理

1. 初始化 P1 口（行输出、列输入，或相反，依软件策略）
2. 依次将某一行（或列）置低，其余行置高
3. 读取列（或行）状态，若有键按下，对应列线为低
4. 行列组合确定键值（如 Row0 + Col2 → 某一固定键码）
5. 需做 **消抖** 处理，避免误触发

### 编程要点

- **显示**：P0 写段码，P2.0/P2.1 控制两位动态扫描（注意反相逻辑）
- **键盘**：P1 低 4 位与 P1 高 4 位配合扫描，建立 16 键键值表
- **综合应用**：读键值后更新显示缓冲区，在主循环或定时中断中刷新数码管

## 图4：AT89C51 四向交通灯控制电路

![四向交通灯控制电路](img/image-9cd32730-661f-472f-824c-c9f75c027119.png)

### 电路概述

本电路构成 **四向十字路口交通灯控制系统**，基于 **AT89C51（U1）** 最小系统（含 12 MHz 晶振与复位电路），用 P0 口低 6 位驱动 12 只 LED（4 个方向 × 红黄绿各 1 组，东西/南北方向各两盏并联）。

### 主要元器件

| 编号 | 型号/类型 | 作用 |
| :--- | :--- | :--- |
| U1 | AT89C51 | 交通灯控制器 |
| X1 | 12 MHz 晶振 | 系统时钟 |
| C1、C2 | 22 pF | 晶振负载电容 |
| C3 | 10 μF | 复位电容 |
| R1 | 10 kΩ | 复位下拉电阻 |
| R2～R13 | 220 Ω | LED 限流电阻 |
| D1～D12 | LED（红/黄/绿） | 四向交通信号灯 |

### 最小系统连接

**振荡电路**

- XTAL1（19）、XTAL2（18）接 12 MHz 晶振 X1
- 晶振两端各经 22 pF 电容（C1、C2）接地

**复位电路**

- RST（9）经 10 μF 电容（C3）接 VCC，经 10 kΩ 电阻（R1）接 GND
- 上电时 RST 产生高电平复位脉冲

**控制引脚**

- EA/VPP（31）接 **VCC**：使用内部 Flash 程序存储器
- PSEN、ALE 未外接

### LED 分组与引脚映射

东西方向（左、右两组 LED 并联）：

| MCU 引脚 | 颜色 | 对应 LED |
| :--- | :--- | :--- |
| P0.0（39） | 红 | D1、D7 |
| P0.1（38） | 黄 | D2、D8 |
| P0.2（37） | 绿 | D3、D9 |

南北方向（上、下两组 LED 并联）：

| MCU 引脚 | 颜色 | 对应 LED |
| :--- | :--- | :--- |
| P0.3（36） | 红 | D10、D4 |
| P0.4（35） | 黄 | D11、D5 |
| P0.5（34） | 绿 | D12、D6 |

每只 LED 阳极接对应 P0 引脚，阴极经 220 Ω 电阻接 GND。

### 工作逻辑

- P0 引脚 **高电平（1）** → 对应 LED **点亮**
- P0 引脚 **低电平（0）** → 对应 LED **熄灭**
- 同一方向红黄绿应 **互斥**：同一时刻只应有一种颜色有效（软件保证）

### 典型灯态与 P0 值

| 交通状态 | 东西方向 | 南北方向 | P0 有效位 | 十六进制 |
| :--- | :--- | :--- | :--- | :--- |
| 东西通行 | 绿 | 红 | P0.2=1, P0.3=1 | 0x0C |
| 南北通行 | 红 | 绿 | P0.0=1, P0.5=1 | 0x21 |
| 东西黄灯 | 黄 | 红 | P0.1=1, P0.3=1 | 0x0A |
| 南北黄灯 | 红 | 黄 | P0.0=1, P0.4=1 | 0x11 |
| 全红（过渡） | 红 | 红 | P0.0=1, P0.3=1 | 0x09 |

（具体编码依位定义，上表按 P0.0～P0.5 分别对应东西红、黄、绿与南北红、黄、绿。）

### 编程要点

- 用状态机实现：绿灯 → 黄灯 → 全红 → 切换方向 → 循环
- 各状态维持时间由定时器或延时函数控制
- 写 P0 前建议先清零再按位赋值，避免多灯同亮冲突
- C51 示例：`P0 = 0x0C;` 表示东西绿、南北红

## 四图对比小结

| 图号 | 核心功能 | 主要使用端口 | 关键外设 |
| :--- | :--- | :--- | :--- |
| 图1 | 单 LED 闪烁 | P0.0 | 排阻 RP1 |
| 图2 | 6 位数码管显示 | P0（段码）、P2（位选）、P1.0（锁存） | 74HC573、RP1 |
| 图3 | 2 位数码管 + 键盘 | P0（段码）、P2（位选）、P1（键盘） | 74HC04、RP1 |
| 图4 | 四向交通灯 | P0.0～P0.5 | 晶振、复位、限流电阻 |

**共性说明**

- AT89C51/80C51 均属于 8051 内核，I/O 编程模型一致
- **P0 口** 为开漏，接段码或 LED 时需 **外部上拉**（图1～图3 使用 RP1；图4 LED 直接驱动，由 MCU 灌/拉电流）
- **P1、P2、P3** 为准双向口，可直接驱动负载（图2 锁存使能、图3 键盘与位选、图4 未使用）
